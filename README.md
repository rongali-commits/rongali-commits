## Working principles

1. **Show what is running.** Live browser implementations and proposed production integrations are labelled separately.
2. **Keep results reproducible.** Public projects include fictional inputs, automated tests, screenshots, and setup instructions.
3. **Design for handover.** The definition of done includes deployment, documentation, and clear operating boundaries.
4. **Protect real data.** Portfolio demos use reserved domains and fictional records; client credentials and data never enter public repositories.
